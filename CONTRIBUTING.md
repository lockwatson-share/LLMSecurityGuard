# Contributing to LLMSecurityGuard

Thank you for your interest in contributing! We welcome contributions of all kinds.  

## Ways to Contribute
- Add new **sanitizers** in the `sanitizer/` folder.
- Add new **LLM adapters** in the `llm_adapters/` folder.
- Improve **risk scoring** logic in `sanitizer/input_sanitizer.py`.
- Enhance the **dashboard** or **logging modules**.
- Add **documentation**, examples, or tutorials.

## Pull Request Guidelines
1. Fork the repository and create your branch (`git checkout -b feature-name`).
2. Commit your changes (`git commit -m 'Add some feature'`).
3. Push to the branch (`git push origin feature-name`).
4. Open a **pull request** with a clear description of the changes.

## Code Style
- Use clear, descriptive names for functions, classes, and variables.
- Add docstrings for new functions or modules.
- Keep code modular and reusable.

---

We appreciate your contributions! Your work helps make LLMSecurityGuard more secure and useful for the community.
